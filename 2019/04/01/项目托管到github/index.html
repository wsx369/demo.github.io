项目托管到github

发表于 2019-04-01 | 分类于 搭建博客

相关步骤

1、安装Node.js和配置好Node.js环境，打开cmd命令行

node -v

打开GitHub官网官网，选择sign up

2、Github账户注册和新建项目，项目必须要遵守格式：账户名.github.io，不然接下来会有很多麻烦。并且需要勾选Initialize this repository with a README

3、安装Hexo，在自己认为合适的地方创个文件夹，我是在D盘建了一个blog文件夹。然后通过命令行进入到该文件夹里面

输入npm install hexo -g，开始安装Hexo

输入hexo -v，检查hexo是否安装成功

输入hexo init，初始化该文件夹（等待时间有点长。。。）

看到后面的“Start blogging with Hexo！”，表示成功了！！！！

输入npm install，安装所需要的组件

输入hexo g，首次体验Hexo

输入hexo s，开启服务器，访问该网址，正式体验Hexo

问题：假如页面一直无法跳转，那么可能端口被占用了。此时我们ctrl+c停止服务器，接着输入“hexo server -p 端口号”来改变端口号

hexo server -p 5000

有图就成功了，简单吧！

4、将Hexo与Github page联系起来，设置Git的user name和email（如果是第一次的话）,打开Git Bash Here,执行如下命令

git config --global user.name 'wsx369'

git config --global user.email '1445407248@qq.com'

上图是在其文件夹里面鼠标右键，点击Git Base Here。这里“feng”可以替换成自己的用户名，邮箱可以替换成自己的邮箱

输入cd ~/.ssh，检查是否由.ssh的文件夹

如果没有找到，输入ssh-keygen -t rsa -C “929762930@qq.com”，连续三个回车，生成密钥，最后得到了两个文件：id_rsa和id_rsa.pub（默认存储路径是：C:\Users\Administrator.ssh）。

输入eval "$(ssh-agent -s)"，添加密钥到ssh-agent

再输入ssh-add ~/.ssh/id_rsa，添加生成的SSH key到ssh-agent

5、登录Github，点击头像下的settings，添加ssh，新建一个new ssh key，将id_rsa.pub文件里的内容复制到key中 Title可以自己随便去名称

6、输入ssh -T git@github.com，测试添加ssh是否成功。如果看到Hi后面是你的用户名，就说明成功了

问题：假如ssh-key配置失败，那么只要以下步骤就能完全解决

首先，清除所有的key-pair

ssh-add -D

rm -r ~/.ssh

删除你在github中的public-key

重新生成ssh密钥对

ssh-keygen -t rsa -C "xxx@xxx.com"

接下来正常操作
在github上添加公钥public-key:
1、首先在你的终端运行 xclip -sel c ~/.ssh/id_rsa.pub将公钥内容复制到剪切板
2、在github上添加公钥时，直接复制即可
3、保存

测试：
在终端 ssh -T git@github.com

7、配置Deployment，在其文件夹中，找到_config.yml文件，修改repo值（在末尾） 

添加repositiry:repo值是你在github项目里的ssh（右下角） Clone or download  Use SSH

repositiry:git@github.com:wsx369/demo.github.io.git

branch:master

8、新建一篇博客，在cmd执行命令：hexo new post “博客名”

hexo new post "你好世界"

在生成以及部署文章之前，需要安装一个扩展：

npm install hexo-deployer-git --save

使用编辑器编好文章，那么就可以使用命令：hexo d -g，生成以及部署了，部署成功后访问你的地址：http://用户名.github.io。那么将看到生成的文章
